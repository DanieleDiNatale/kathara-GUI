Introduzione a Kathara

Kathara è un emulatore di rete basato su Docker che permette di creare scenari di rete realistici. È Utilizzato solo per questo corso "SPECIALE" XD.

Struttura base: Ogni lab è una cartella con file di configurazione.

🖥️ Interfaccia Grafica (GUI)

La GUI permette di creare topologie di rete visivamente e esportarle per Kathara.

Dispositivi Disponibili

  • PC - Computer
  • Router - Dispositivo di routing (colore azzurro)
  • Switch - Commutatore di rete (colore arancione)
  • Hub - Concentratore di rete (colore rosso)
  • Cloud - Connettività esterna (colore viola)

Comandi da Console

Puoi usare i comandi testuali nella sezione "Console Commands":

# Aggiungi dispositivo
add router R1
add pc PC1
add switch SW1

# Collega dispositivi (cavi: copper-straight, copper-cross, fiber, serial, phone, coaxial)
connect PC1 R1 copper-straight
connect PC2 SW1

# Configura IP (eth0-eth3)
ip PC1 eth0 192.168.1.10 192.168.1.254

# Elimina dispositivo
del PC1

# Lista topologia
list

# Aiuto
help

Funzionalità

  • SET IP: Configura indirizzo IP e gateway
  • PING: Testa connettività direttamente dalla GUI
  • EXPORT: Esporta file di configurazione Kathara
  • START/STOP: Avvia/ferma i container
  • CONNECT: Connettiti a un dispositivo

📁 Struttura di un Lab

mio_lab/
├── lab.conf          # Topologia della rete
├── PC1.startup       # Configurazione PC1
├── Router1.startup   # Configurazione Router1
└── ...

📝 Esercizio 1: Rete Semplice (2 PC)

Situazione

Creare una rete con 2 PC collegati direttamente via cavo cross-over.

lab.conf

PC1[0]=A
PC2[0]=A

PC1.startup

ip addr add 192.168.1.1/24 dev eth0

PC2.startup

ip addr add 192.168.1.2/24 dev eth0

Verifica: Ping da PC1 a PC2 (e viceversa)

📝 Esercizio 5: Rete con 2 Router

Situazione

2 Router collegati tra loro, ciascuno con un PC. Rete Router1: 192.168.1.0, Rete Router2: 192.168.2.0

lab.conf

PC1[0]=A
Router1[0]=A
Router1[1]=B
Router2[0]=B
Router2[1]=C
PC2[0]=C

PC1.startup

ip route add default via 192.168.1.254

PC2.startup

ip addr add 192.168.2.1/24 dev eth0
ip route add default via 192.168.2.254

Router1.startup

ip addr add 192.168.1.254/24 dev eth0
ip addr add 10.0.0.1/24 dev eth1

Router2.startup

sysctl -w net.ipv4.ip_forward=1
ip addr add 192.168.2.254/24 dev eth0
ip addr add 10.0.0.2/24 dev eth1

⌨️ Comandi Utili

Gestione Lab

# Avvia lab
kathara lstart -d nome_lab

# Ferma lab
kathara lstop -d nome_lab

# Rimuovi lab
kathara lclean -d nome_lab

# Lista container
kathara list

# Collegati a dispositivo
kathara connect -d nome_lab -n nome_dispositivo

# Esegui comando
kathara exec -d nome_lab -n nome_dispositivo "comando"

Debug

# Test connettività
ping <indirizzo_ip>

# Visualizza routing
ip route show
ip addr show

# Trace route
traceroute <destinazione>

# Cattura traffico
tcpdump -i eth0

# Visualizza tabella ARP
arp -a

# Test DNS
nslookup <dominio>

💡 Regole importanti

  • Cavo Straight-Through: PC-Router, PC-Switch, PC-Hub
  • Cavo Cross-Over: PC-PC diretto, Router-Router
  • Router: Deve sempre avere sysctl -w net.ipv4.ip_forward=1
  • Default gateway: I PC devono avere il route verso il router
  • Stessa rete: Dispositivi sulla stessa lettera (A, B, C) comunicano
  • Rete diversa: Serve un Router per comunicare tra reti diverse
